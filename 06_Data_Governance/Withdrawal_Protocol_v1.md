# Withdrawal Protocol v1

## Purpose
This document defines the early withdrawal logic for participants in QashqAI Voice activities.

## Core principle
Participants must have a meaningful way to withdraw their contribution.

## Withdrawal rights
A participant should be able to request withdrawal of:
- recordings
- metadata
- related documentation
- future use permissions where applicable

## Minimum withdrawal process
1. receive the request
2. confirm the identity or participant ID where possible
3. identify relevant files or materials
4. document the request
5. remove or restrict materials as appropriate
6. confirm action taken

## Ethical requirements
- the process should be understandable
- the process should not punish participation
- the participant should be treated respectfully
- any limits must be explained clearly
- records of withdrawal action should be kept

## Risks
- unclear file tracking
- incomplete documentation
- confusion between archived and active materials
- failure to communicate limits honestly

## Current note
This is an early working withdrawal structure and should later be translated into participant-facing language.
