# Consent Framework v1

## Purpose
This document defines the early consent logic for QashqAI Voice.

## Core principle
No recording, documentation, or language data should be collected without clear and informed consent.

## Consent requirements
- participation must be voluntary
- consent must be understandable
- the purpose of collection must be explained
- possible future uses must be explained in clear language
- the participant must be able to refuse without pressure
- the participant must be able to ask questions
- consent should be documented

## Minimum consent elements
- participant identity or participant ID
- date of consent
- language of explanation
- type of material being collected
- intended purpose
- storage logic
- review possibility
- withdrawal option

## Ethical rules
- avoid pressure or confusion
- do not collect more than necessary
- respect cultural sensitivity
- do not treat language data as extractive raw material
- maintain human oversight

## Current status
This is an early working framework and should later be expanded into a more formal project consent document.
