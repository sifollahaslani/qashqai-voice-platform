# QashqAI Voice Export Review Checklist

## Purpose

Use this checklist before any QashqAI Voice material is shared outside local project custody. Export includes sending files to researchers, institutions, archives, cloud services, public repositories, private collaborators, or AI providers.

## Before Export Checklist

### 1. Item Identification

- [ ] `item_id` is known.
- [ ] `recording_id` is known.
- [ ] Metadata file exists.
- [ ] Consent record exists.
- [ ] Validation record exists.
- [ ] Export purpose is documented.

### 2. Consent Review

- [ ] Consent is active.
- [ ] Consent has not been revoked or narrowed.
- [ ] Access level permits the proposed export.
- [ ] Institutional sharing is allowed, if applicable.
- [ ] Research use is allowed, if applicable.
- [ ] Public release is allowed, if applicable.
- [ ] Commercial use is not implied unless explicitly allowed.
- [ ] AI use is not implied unless explicitly allowed.

If any field is missing or ambiguous, stop and keep the item private.

### 3. Cultural Validation Review

- [ ] Cultural validation is complete.
- [ ] Sensitivity level permits export.
- [ ] Names and places have been checked.
- [ ] Translation uncertainty has been reviewed.
- [ ] Dialect notes are preserved where relevant.
- [ ] Publication recommendation matches the export purpose.
- [ ] AI-use recommendation matches the export purpose, if relevant.

### 4. File Selection

- [ ] Only necessary files are included.
- [ ] Raw audio is excluded unless specifically approved.
- [ ] Identity records are excluded unless specifically approved.
- [ ] Private notes are excluded.
- [ ] Temporary processing files are excluded.
- [ ] Embeddings, indexes, and model outputs are excluded unless specifically approved.

### 5. Recipient and Terms

- [ ] Recipient is identified.
- [ ] Purpose is documented.
- [ ] Retention expectations are documented.
- [ ] Redistribution restrictions are documented.
- [ ] AI-use restrictions are documented.
- [ ] Revocation or takedown expectations are documented.
- [ ] Attribution or anonymity requirements are documented.

### 6. Export Logging

- [ ] Export manifest is created.
- [ ] Export log row is created.
- [ ] Audit log entry is created.
- [ ] Consent snapshot is preserved where appropriate.
- [ ] Validation snapshot is preserved where appropriate.

## Stop Conditions

Do not export if:

- consent is missing, revoked, or unclear;
- validation is pending or restricted;
- the recipient cannot accept restrictions;
- raw audio is requested without explicit permission;
- the export would enable AI training or dataset creation without explicit permission;
- there is unresolved reviewer disagreement;
- narrator or family concerns are unresolved.

## Minimal Export Record

Record:

- export ID;
- item ID;
- recording ID;
- recipient;
- purpose;
- files included;
- files excluded;
- consent ID and status;
- validation ID and status;
- restrictions;
- responsible person;
- date.
