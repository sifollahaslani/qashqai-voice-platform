# Data Pipeline Notes

## Objective
Define a simple and responsible data pathway for future QashqAI Voice language work.

## Proposed early pipeline
1. speaker identification and consent
2. voice recording collection
3. metadata registration
4. secure storage
5. file organization and naming
6. transcription preparation
7. review and validation
8. future model-ready formatting

## Early data types
- audio recordings
- speaker consent records
- speaker metadata
- transcription notes
- translation notes
- project logs

## Minimum metadata fields
- speaker ID
- date
- location
- dialect/community note
- language
- consent status
- recording topic
- reviewer note

## Governance requirements
- consent must be explicit
- withdrawal must be possible
- sensitive data must be minimized
- community context must be respected
- storage must be organized and reviewable

## Risks
- collecting data without full structure
- weak naming conventions
- unclear consent records
- loss of source context
- future incompatibility with technical workflows

## Next step
Create a practical naming and storage standard for all future recordings and related documentation.
