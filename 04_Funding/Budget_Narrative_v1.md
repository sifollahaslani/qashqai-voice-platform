# Budget Narrative v1

## Purpose
This document explains how early-stage funding for QashqAI Voice would be used and why the budget supports a realistic MVP phase.

## Budget philosophy
The early budget is designed to be:
- lean
- practical
- transparent
- mission-aligned
- appropriate for a first structured phase

## Main budget areas
### 1. Documentation and project preparation
Resources are needed to prepare institutional materials, organize project files, define scope, and create coherent planning documents.

### 2. Governance and consent development
A serious language project requires clear ethical and governance foundations. This includes time and effort for:
- consent logic
- withdrawal rules
- data handling structure
- cultural responsibility safeguards

### 3. Data preparation logic
Even before technical scaling, the project must define a usable and reviewable structure for:
- recording organization
- metadata handling
- naming standards
- transcription preparation

### 4. Technical setup
Initial technical support may include:
- local development environment setup
- preprocessing logic
- early testing scripts
- documentation of technical pathways

### 5. Outreach and institutional engagement
The project also requires preparation for dialogue with:
- universities
- funders
- cultural institutions
- research partners

## Why a modest early budget matters
A small early budget can create disproportionate value by turning a vulnerable idea into a credible institutional and technical foundation.

## Expected outcome of budget use
By the end of the early funded phase, QashqAI Voice should have:
- a documented MVP structure
- core institutional materials
- funding-facing documents
- basic governance files
- a clearer technical and cultural execution pathway
